# 排查会话文件模板

## 问题驱动排查（模式 A）

文件命名：`YYYY-MM-DD_问题关键词.md`（如 `2026-02-21_翻页闪烁.md`）

```markdown
# 问题排查：[问题简述]

- 日期：YYYY-MM-DD
- 基线 commit：`SHA` (commit message)
- 问题描述：[用户原始描述]
- 涉及区域：[区域编号] [区域名]

## 排查过程

### 1. [排查的第一个文件/模块]

文件：`路径`

[排查内容和发现]

### 2. [排查的第二个文件/模块]

文件：`路径`

[排查内容和发现]

## 发现

- 🐛 [P1] [问题描述] — 已确认 / 待确认
- ⚠️ [P2] [问题描述] — 待确认

## 根因分析

[根因分析，或"需要运行时验证"的说明]

## 结论

[总结和后续建议]
```

## 系统性巡查（模式 B）

文件命名：`YYYY-MM-DD_区域编号_区域名.md`（如 `2026-02-21_R4.1_TTS引擎.md`）

```markdown
# 系统巡查：[区域编号] [区域名]

- 日期：YYYY-MM-DD
- 基线 commit：`SHA` (commit message)
- 排查区域：[区域编号] [区域名]
- 核心文件：（从 area-map 复制）

## 排查过程

### 检查：[检查点名称]

文件：`路径`

- ✅ [正常项描述]
- ⚠️ [P2] [问题描述] — 待确认
- 🐛 [P1] [问题描述] — 已确认

## 发现

（汇总所有问题）

## 结论

（区域质量评估）
```
