# Glint

English · [中文](README.md)

A polished macOS terminal made for AI agents. SwiftUI + AppKit, with [Ghostty](https://ghostty.org) under the hood.

## Install

### Homebrew (recommended)

```bash
brew tap chenbstack/glint
brew install --cask glint
```

The tap repo lives at `github.com/chenbstack/homebrew-glint`. Its Cask formula is generated by the release workflow — see [Releasing](#releasing).

### Manual download

Grab the latest `Glint-x.y.z.dmg` from the [Releases](https://github.com/chenbstack/glint/releases) page, mount it, and drag `Glint.app` into `/Applications`.

If macOS refuses to open it with "developer cannot be verified", the build is not yet Apple-notarized. Run once:

```bash
xattr -dr com.apple.quarantine /Applications/Glint.app
```

The Homebrew Cask does this for you automatically.

## Upgrade

```bash
brew upgrade --cask glint
```

If you enabled auto-update in Glint's settings, Sparkle will prompt you on launch when a new version ships.

## Uninstall

```bash
brew uninstall --cask glint
```

This also clears `~/Library/Application Support/Glint` and your preferences.

## Build from source

Requires Xcode 16+ and macOS 14+.

```bash
git clone https://github.com/chenbstack/glint.git
cd glint
# Fetch the Ghostty xcframework (~500 MB, not committed).
# Host it yourself or point GHOSTTYKIT_URL at a release asset.
export GHOSTTYKIT_URL='https://…/GhosttyKit.xcframework.tar.gz'
bash scripts/fetch-ghosttykit.sh
open Glint.xcodeproj
```

## Releasing

Tag a commit `vX.Y.Z` on `main`. The `.github/workflows/release.yml` workflow then:

1. Stamps the version into `Info.plist`.
2. Builds a Release archive and exports `Glint.app`.
3. (Optional) Codesigns with Developer ID and notarizes through `notarytool`.
4. Builds a `.dmg`.
5. (Optional) Signs the dmg with Sparkle's EdDSA key and appends an `<item>` to `appcast.xml`.
6. Publishes a GitHub Release with the dmg + appcast.
7. Pushes the updated `appcast.xml` back to `main`.

### Required secret

| Secret | Purpose |
|---|---|
| `GHOSTTYKIT_URL` | Public URL of `GhosttyKit.xcframework.tar.gz` (or `.tar.xz` / `.zip`). |

### Optional secrets — enable signing + Sparkle

Skip these and you get an unsigned (ad-hoc) build. Users still install it via Cask thanks to the `postinstall` `xattr -dr com.apple.quarantine` trick. Add them when you have an Apple Developer account ($99/yr).

| Secret | Purpose |
|---|---|
| `APPLE_CERT_P12_BASE64` | `base64` of your Developer ID Application `.p12`. |
| `APPLE_CERT_PASSWORD` | Password set when exporting the `.p12`. |
| `APPLE_NOTARY_ID` | Apple ID used for notarization. |
| `APPLE_NOTARY_PASSWORD` | App-specific password from appleid.apple.com. |
| `APPLE_NOTARY_TEAM_ID` | 10-char team identifier. |
| `SPARKLE_ED_PRIV_KEY` | EdDSA private key from Sparkle's `generate_keys`. The public counterpart goes into `Info.plist` as `SUPublicEDKey`. |

### Sparkle key bootstrap

```bash
# Once, locally:
curl -fsSL https://github.com/sparkle-project/Sparkle/releases/download/2.6.4/Sparkle-2.6.4.tar.xz | tar -xJ
./Sparkle-2.6.4/bin/generate_keys
# Stores the private key in your Keychain; prints the public key.
# Copy the private key into the SPARKLE_ED_PRIV_KEY repo secret.
# Paste the public key into Glint/Resources/Info.plist as SUPublicEDKey.
```
